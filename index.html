WAPT-04 — Exposed .git Directory Disclosure
Latar Belakang
Pada periode 11–24 Januari 2025, dilakukan pengujian penetrasi terhadap aplikasi web DIBIMBING SHOP dalam lingkungan laboratorium Cyber Security Bootcamp Dibimbing.id. Pengujian menggunakan metode black-box testing berdasarkan OWASP WSTG v4.2 untuk mengidentifikasi dan memvalidasi kerentanan keamanan pada sisi user dan admin. Audit ini bertujuan melindungi data sensitif pelanggan, mencegah manipulasi transaksi, dan mengidentifikasi risiko bisnis akibat eksploitasi kerentanan.
Scope & Objective
Ruang Lingkup: Fitur pengguna (guest / unauthenticated user), panel administrator (authenticated user), validasi input dan output aplikasi, mekanisme autentikasi dan manajemen sesi, serta kontrol keamanan fitur administratif.

Fokus: Identifikasi dan validasi kerentanan aplikasi web dari perspektif Red Team menggunakan pendekatan black-box dan grey-box sesuai Rules of Engagement.
Main Findings
Critical
WAPT-01 — SQL Injection pada cart.php (Parameter Coupon Code)
Injeksi SQL pada parameter coupon di cart.php memungkinkan attacker membaca seluruh isi database server. Error MySQL terekspos langsung di halaman: "You have an error in your SQL syntax...". Dikonfirmasi menggunakan Burp Suite dan SQLmap.
Mitigasi: Prepared Statements dan strict input validation.
Critical
WAPT-02 — Hardcoded Credentials (.env Exposure via GitHub)
Kredensial database tersimpan langsung di source code karena tidak menggunakan environment variable. Repository GitHub dapat diakses publik sehingga username, password database, dan konfigurasi sensitif terekspos sepenuhnya.
Mitigasi: Gunakan environment variables dan hapus credentials dari source code.
High
WAPT-03 — Stored Cross-Site Scripting (XSS) di Admin Panel (Product URL)
Stored XSS pada fitur Product URL di Admin Panel. Script berbahaya tersimpan di database dan dieksekusi browser setiap pengguna yang mengakses halaman, berpotensi mencuri session token administrator.
Mitigasi: Input sanitization, output encoding, dan Content Security Policy (CSP).
High
WAPT-04 — Exposed Git Repository (.git Directory Disclosure)
Direktori .git dapat diakses publik melalui web browser, mengekspos seluruh source code, history commit, konfigurasi server, dan informasi sensitif. Remote origin mengarah ke repository GitHub publik.
Mitigasi: Block akses ke direktori sensitif di web server dan hapus .git dari web root.
Rekomendasi Keamanan
Meningkatkan kontrol keamanan aplikasi melalui prepared statements dan validasi input ketat. Mengamankan kredensial dengan environment variables dan hashing. Memastikan komunikasi terenkripsi HTTPS/TLS. Memperkuat autentikasi dan manajemen sesi. Melakukan evaluasi keamanan berkala melalui vulnerability assessment dan penetration testing.
Tools & Environment
Kali Linux Burp Suite SQLmap Nmap OWASP WSTG v4.2 XSS Hunter Dibimbing Shop (Lab)

IR-04 — Persistence & Privilege Escalation Attempt
Latar Belakang
Pada periode 22 Februari – 7 Maret 2025, dilakukan analisis insiden keamanan pada server Linux publik dalam lingkungan laboratorium Cyber Security Bootcamp Dibimbing.id. Analisis dilakukan menggunakan pendekatan Threat Hunting dan SIEM Wazuh untuk mengidentifikasi brute force SSH, akses tidak sah, eskalasi hak akses, dan indikasi persistence. Simulasi ini bertujuan memahami bagaimana attacker melakukan password guessing terhadap SSH dan bagaimana aktivitas tersebut dapat dideteksi melalui analisis log.
Scope & Objective
Ruang Lingkup: Log autentikasi SSH, log sistem Linux, aktivitas jaringan terkait akses mencurigakan, dan alert keamanan dari SIEM Wazuh. Analisis dilakukan secara forensik menggunakan teknik log correlation dan event reconstruction.

Fokus: Mengidentifikasi aktivitas brute force, akses tidak sah, dan eskalasi hak akses pada sistem target menggunakan pendekatan Blue Team dan Threat Hunting.
Main Findings
Critical
IR-01 — SSH Brute Force Attack
Aktivitas brute force terhadap layanan SSH terdeteksi melalui ratusan percobaan login gagal dalam interval waktu singkat. Seluruh percobaan berasal dari IP 180.243.121.188, pertama terdeteksi pada 4 Maret 2025 pukul 12:04 UTC. Grep pada /var/log/auth.log* menampilkan ratusan entri "Failed password for root from 180.243.121.188".
Mitigasi: Implementasi fail2ban, rate limiting SSH, dan blokir IP mencurigakan.
Critical
IR-02 — Successful SSH Login sebagai Analyst (Akses Awal)
Attacker berhasil memperoleh akses awal pada akun dengan hak akses terbatas (analyst) setelah serangkaian percobaan autentikasi. Login berhasil pertama kali pada 6 Maret 2025 pukul 13:29 UTC dari IP 180.243.121.188. Log menunjukkan beberapa entri "Accepted password for analyst from 180.243.121.188" pada rentang waktu yang berdekatan.
Mitigasi: Autentikasi SSH key, nonaktifkan password-based login.
Critical
IR-03 — Eskalasi Hak Akses ke Root (Full System Compromise)
Setelah berhasil login sebagai analyst, attacker berhasil meningkatkan hak akses menjadi root pada 6 Maret 2025 pukul 16:31 UTC. Log menunjukkan "Accepted password for root from 180.243.121.188". Aktivitas root berlanjut hingga 7 Maret 2025 pukul 02:39 UTC, mengindikasikan full system control dan persistence oleh attacker.
Mitigasi: Nonaktifkan login root langsung via SSH, implementasi Multi-Factor Authentication.
High
IR-04 — Aktivitas Autentikasi Mencurigakan: Potensi Persistence & Privilege Escalation
Aktivitas autentikasi mencurigakan menunjukkan potensi persistence dan kontrol penuh terhadap sistem. Ditemukan percobaan eskalasi hak akses pada 7 Maret 2025 pukul 03:15 UTC dengan evidence sebagai berikut:

• sudo: www-data : user NOT in sudoers ; USER=root COMMAND=/usr/bin/su
• su[353196]: FAILED SU (to root) www-data
• sudo: www-data : user NOT in sudoers ; USER=root COMMAND=/usr/bin/less /etc/shadow

Analisis korelasi log melalui Wazuh SIEM pada 7 Maret 07:30 UTC merekonstruksi timeline lengkap: 742 total alerts, 597 authentication failures, 5 authentication successes.
Mitigasi: Hardening konfigurasi sudo, real-time SIEM monitoring, server hardening berkala.
Dampak Risiko
Jika serangan tidak terdeteksi, attacker berpotensi mengambil alih kendali penuh server, mengakses atau mencuri data sensitif, menyisipkan malware atau backdoor, dan menyalahgunakan sumber daya sistem. Kompromi server publik juga dapat menyebabkan kerusakan reputasi organisasi serta gangguan operasional yang signifikan.

Security Recommendations

🛡️
Meningkatkan kontrol keamanan aplikasi melalui penerapan prepared statements dan validasi input yang ketat untuk mencegah SQL Injection.
🔐
Mengamankan kredensial sensitif dengan environment variables dan hashing — jangan hardcode di source code atau repository publik.
🔒
Memastikan seluruh komunikasi aplikasi menggunakan koneksi terenkripsi (HTTPS/TLS) untuk mencegah kebocoran data.
🔑
Memperkuat autentikasi SSH dengan key-based authentication. Nonaktifkan login root langsung dan password-based login.
🚫
Implementasi fail2ban dan pembatasan percobaan login otomatis untuk mencegah serangan brute force pada SSH dan web aplikasi.
📊
Monitoring log real-time menggunakan SIEM dan audit keamanan berkala melalui vulnerability assessment dan penetration testing.
